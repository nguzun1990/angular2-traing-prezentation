Shared Modules

  • exists to make commonly used components, directives, and pipes available for use in many other modules
  • Don't specify providers here!

Core Module

  • Specify the Core items that will be used in all modules

Routing module

  • Separates routing / application concerns
  • Provides a module to replace or remove when testing the application
  • Provides a well-known location for routing service providers including guards and resolvers
  • Does not declare components

Dependency Injection

  • DI -> software design pattern used for resolving dependencies
  • Dependency -> an object that can be used (a service)
  • Injection -> passing of a dependency to a dependent object (a client)

A class receives its dependencies from external sources rather than creating them itself.

Resolvers / pre-fetching component data

                            
@Injectable()
export class DataResolver implements Resolve«SomeModel» {
  constructor(private cs: DataService, private router: Router) {}
  resolve(route: ActivatedRouteSnapshot, state: RouterStateSnapshot): Promise«SomeModel» {
    let id = route.params['id'];
    return this.cs.getData(id).then(data => {
      if (data) {
        return data;
      } else { // id not found
        this.router.navigate(['/another-url']);
        return null;
      }
    });
  }
}

Asynchronous routing

  • Lazy loading route configuration
  • Preloading - Custom preloading strategies: 
                                
{
  path: 'url-to-load',
  loadChildren: 'app/module-name/module-name.module#ModuleNameModule',
  data: { preload: true }
},
                            
                        
                            
export class SelectivePreloadingStrategy implements PreloadingStrategy {
  preloadedModules: string[] = [];

  preload(route: Route, load: () => Observablee«any»): Observablee«any» {
    if (route.data && route.data['preload']) {
      this.preloadedModules.push(route.path);
      return load();
    } else {
      return Observable.of(null);
    }
  }
}

PropertyDescription
myControl.value

the value of a FormControl.

myControl.status

the validity of a FormControl. Possible values: VALID, INVALID, PENDING, or DISABLED.

myControl.pristine

true if the user has not changed the value in the UI. Its opposite is myControl.dirty.

myControl.untouched

true if the control user has not yet entered the HTML control and triggered its blur event. Its opposite is myControl.touched.
